The tFormer plugin allows you to easily and unobtrusively control your HTML forms.

Powerful HTML form validation and control.

Download ~19KB

Intuitive validation rules. User friendly control of error/success reports, submit, and other form processes.


Powerful

Long list of validation rules. Useful methods(). Every part of tFormer developed to make form validation and control as good as can be imagined.

Simple and logic rules

Everybody knows what `<`, `>`, `@` or `*` means. You don't need to write huge amount of data attributes, just one rules string like '* @ l>10' to understand how to validate the field.

See full validation rules list.

User friendly

User Interface is the main part of all types of apps. tFormer know that and can interact with user exactly how developer wants.

See examples.

Highly customizable

All class names, timeouts, and even validation function can be customized by developer. Easily and smoothly.

See details in options documentation.

Independent

The plugin is totally independent. No jQuery or other external lib needed - download -> include -> use.

See details in installation and usage section.

Easily themable

With common, process, disabled and error class names you can customize any type of user interaction with form.

Take a look on examples.

var request_example = tFormer( 'request_example', {
	fields: {
		field_name: {
			url   : 'ajax.html',
			method: 'post',
			data  : { // this params will be in post request
				custom_param: 'custom value',
				twitter     : '@TjRus'
			},
			start : function () {}, // function executed when ajax request starts
			end   : function ( result ) { // function executed when ajax request ends
				// `result` is what your request returns
				// here you should decide is the field valid or no and return true/false.
				return false;
			}
		}
	},
	submit: function ( event ) {
		if ( event.preventDefault ) {
			event.preventDefault();
		} else {
			event.returnValue = false;
		}
		// rewriting default submit code
		alert( 'Your request form verified successfully.' );
		request_example.processingOff();
		return false;
	}
} );

Features

  1. No jQuery needed - plugin is absolutely independent; (Tweet this awesome feature)
  2. Long list of validate rules;
  3. Rules are short and intuitive - easy to learn, remember and use;
  4. Independent validation object (can be used for validation separately without main tFormer object);
  5. Own validate method (you can customize it for each field);
  6. Define validate rules in data-attribute or in tFormer options;
  7. Changing rules programmatically with setRules method
  8. Changing options programmatically with set method
  9. Controlling submit button;
  10. Processing control;
  11. Field check button;
  12. `OnFly` validation (on keyup);
  13. Validation timeouts;
  14. onerror & onvalid field handler;
  15. before validation handler;
  16. Validations with AJAX request;
  17. Request validation timeout;
  18. Custom submit function;
  19. Fields error and processing class customization;
  20. ... more to come ...

Methods

validateForm( event )
permalink

Method for validating form.

event - original event of form submit, or {Boolean} true or false.
If event not defined of false form will be validated without displaying errors visually. If defined as original submit event or true - fields with errors in validation will receive error class (errorClass option).

return true if current form passes all validations that defined, otherwise - false.

validateField( *form_field, show_error, no_timeout )
permalink

Method for validating defined form field.

*form_field - original HTML element of form or field name.

show_error - {Boolean}, defines should the script add errorClass to field if not valid or not (false - no visual errors displayed).

no_timeout - {Boolean}, tells the script should it validate the field with defined timeout option or immediately.

return true if current field passes validation, otherwise - false.

toObject()
permalink

Method for getting javascript object with form element names and values.

return { field_name: field_value, ... }

execute( *func_name, *this_el, params )
permalink

Method for executing defined field functions. (You can define them in fields options)

*func_name - {String}, name of function you want to execute

*this_el - HTML element that will be this (context) in executed function

params - {Array}, list of function arguments.

return the result of executed function.

get( *option_name, field_name )
permalink

Method for get form options.

*option_name - {String}, name of option.

field_name - {String}, field name.

return option value. If field_name defined - return related option value. If there is no such option for the field, or field_name no defined - returns global option value.

set( options, field_name )
permalink

Method for setting form options.

*options - {Object}, list of options to set.

field_name - {String}, field name. All options will be set to one field. If not defined - options will be set globally.

setRules( *rules, *field_name, show_errors )
permalink

Method for setting rules to the field.

*rules - {String}, string with rules.

*field_name - {String}, field name, that will receive new rules.

show_errors - {Boolean}, display errors if the field is not valid with new rules.

submitButtonControl( is_valid )
permalink

Method for turning enabling/disabling submit button. (adding/removing disabledClass)

is_valid - {Boolean}, true to enable button, false - disable

submitButtonOn()
permalink

Method for turning enabling submit button. (removing disabledClass)

Alias to submitButtonControl( true )

submitButtonOff()
permalink

Method for turning disabling submit button. (adding disabledClass)

Alias to submitButtonControl( false )

processing( show )
permalink

Method for turning on/off processing for submit button. (adding/removing processingClass)

show - {Boolean}, true to turn on processing, false - off

processingOn( )
permalink

Method for turning on processing for submit button. (adding processingClass)

Alias to processing( true )

processingOff( )
permalink

Method for turning off processing for submit button. (removing processingClass)

Alias to processing( false )

errorControl( *field, *errors, show_error )
permalink

Method that controls displaying field errors, disabling/enabling submit button and firing onerror / onvalid functions.

*field - HTML form field element.

*errors - {Array}, array of errors. Empty array means no errors

*show_error - {Boolean}, should we display errors (add errorClass to field) and fire onerror / onvalid functions or not. If false - errors won't be displayed and onerror / onvalid won't be fired.

lock( )
permalink

Method for locking form. (adding disabledClass to submit button)

unlock( )
permalink

Method for unlocking form. (removing disabledClass from submit button)

init( )
permalink

Method for initializing tFormer object (runs automatically while creating tFormer object). Can be used for reinitializing tFormer object after destroy()

destroy( )
permalink

Method for destroying tFormer object.

Options

errorClass :string('error') permalink

Defined errorClass will be added to field classList if the validation returns false.

processingClass :string('processing') permalink

Defined processingClass will be added to submit button (submitButton) classList when yser submits the form.

disabledClass :string('disabled') permalink

Defined disabledClass will be added to submit button (submitButton) classList if the form elements has validation errors.
When all form fields validation returns true, disabledClass will be automatically removed form submit button.

timeout :number(0) permalink

timeout option defines the delay before validating field only if.

requestTimeout :number(2000) permalink

requestTimeout option defines timeout before validation field with request validation.
If timeout option defined, than request validation will be delayed on a requestTimeout - timeout if the result is more than 0, otherwise it will be ignored.

validateEvent :string('keyup') permalink

This options defines field event for starting validation.

submitButtonControl :boolean(true) permalink

This options tell the plugin should it controll submit button or not.

submitButton :boolean(true) permalink

This options tell the plugin what element is a submit button.

If it's not defined and submitButtonControl defined as true, the plugin will find the first element with type="submit" and will use it as the submit button.

submit :function( event ) permalink

Function for submitting form.

If submit option not defined - form will be submitted as usual HTML form element. Will fire only when all fields of form successfully validated.

event is original form submit event.

this here is current HTML form element.

// defining submit function
submit: function(event) {
	// 1. Disable default submit
	event.preventDefault()
	// 2. your submit code
	// 3. finish
	return false;
}
before :function() permalink

Function will be executed just before each validation of the field.

this here is current HTML field element.

onerror :function() permalink

Function will be executed each time when field validation will return false.

this here is current HTML field element.

onvalid :function() permalink

Function will be executed each time when field validation will return true.

this here is current HTML field element.

fields :object({ field_name: {/*field_options*/}, ... }) permalink

Describes validation and control options for each field.

Fields properties should be a list of objects, named exactly like form elements and contain all needed options.



Available fields options:

rules permalink
:string String with rules for validation
before permalink
:function() Like a global before function.

Fires only for field in which it is defined.
Has higher priority than the same function defined globally.
onerror :function() permalink

Like a global onerror function.

Fires only for field in which it is defined.
Has higher priority than onerror function defined globally.
onvalid :function() permalink

Like a global onvalid function.

Fires only for field in which it is defined.
Has higher priority than onvalid function defined globally.
url :string permalink

URL for request validation. default value gets from

window.location.href

Used only for request validation.
data :object({}) permalink

Data that will be send with AJAX request. The structure of this object should be:

{
	param_name: param_value,
	// other params
}
method :string('get') permalink

Request method: POST or GET

Important only for request validation.
start :function() permalink

Function that will fires when validation request will sent.

this here is current HTML field element.

end :function(response) permalink

Function that will fires when validation request result arrives.

response here is the returned result of your request.

this here is current HTML field element

!important!
In the request validation end(response) function you can do whatever you want with the response, but at the end of funtion your have to return true or false as the result of validation.
own :function() permalink

Function for own validation (ignores all defined rules).

this here is current HTML field element.

Should return true or false (depends on what you decide whether the field valid or not).

Validator

Validator - is separate object (_v_) that used by tFormer for validating form elements values.

You can use it even without main tFormer plugin.

Example of use:
JavaScript _v_
(click to see the code)
// this is how to use validator object separately
_v_('').required(); // => false
_v_('some value').required(); // => true
_v_('some value').validateWithRules('*'); // => true


// basic usage
_v_('value to validate').validationMethod(options);

// Examples
_v_('i@tjrus.com').isValidEmail(); // => true
_v_('i@..tjrus.com').isValidEmail(); // => false

_v_(89).lessThen(100); // => true
_v_(89).validateWithRules('<100'); // => true
_v_(10).matchesTo([10,13]); // => true
_v_(10).validateWithRules('=[10,13]'); // => true

Validation rules

Rules can be combined as a simple string and separated by space or other defined character.
Example of rules usage:
HTML
JavaScript
(click to see the code)
// this is how to defined rules in JavaScript
var my_tformer_object = tFormer('my_form_id', {
	fields: {
		// if the field has only rule option
		test_required: '*', // rules here

		// if field has multiple options
		test_email_required: {
			rules      : '* @', // rules here
			timeout    : 1000,
			errorClass : 'custom_error_class'
		}
	}
});

// this is how to define rules in HTML element for tFormer
<form id="my_form_id">
	<input name="test_required" data-rules="*" />
	<input name="test_required_and_other" data-rules="* a l>=10" />
</form>
`*` required() permalink

Means that the value shouldn't be empty. Required rule have to be at the beginning of rules string.

`@` isValidEmail() permalink

Means that the value should be valid email address.

`@s` isValidEmails() permalink

Means that the value should be valid email addresses, separated by `,`.

`ip` isValidIP() permalink

Valid IP address.

`b64` isValidBase64() permalink

Validate Base64 string.

`url` isValidUrl() permalink

Validate URL.

validation with regular expression.

`c` isValidCard() permalink

All Visa card numbers start with a 4. New cards have 16 digits. Old cards have 13

All MasterCard numbers start with the numbers 51 through 55. All have 16 digits

American Express card numbers start with 34 or 37 and have 15 digits

Discover card numbers begin with 6011 or 65. All have 16 digits

Month Name N (full name) n (abbr)
Day of Month D (1 or 2 digits) d (1 or 2 digits)
Day Name W (full name) w (abbr)
Hour (1-12) H (1 or 2 digits) h (1 or 2 digits)
Hour (0-23) R (1 or 2 digits) r (1 or 2 digits)
Minute I (1 or 2 digits) i (1 or 2 digits)

AM/PM A (upper case) a (lower case)
`request` --- permalink

Tell the script that there should be validation with AJAX request.

Required a few fields options to be set in tFormer config: url, end()

(!) Request validation is not a part of _v_ object.
